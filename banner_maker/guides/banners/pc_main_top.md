# 01. PC Banner — 메인 탑 배너 (Main Top Banner)

> **스코프**: 알바몬(ALBAMON) 메인 페이지 **PC 좌측 상단**에 노출되는 광고 배너의 제작 규격. 광고주가 입고하는 외부 소재(이미지 1장)의 사이즈·여백·카피·폰트·파일 마감 규칙 전체를 다룬다.
> **노출 매체**: 알바몬 PC 메인
> **상위 가이드**: `배너 공통 제작가이드` (별도 문서) — 본 문서는 메인 탑 배너 **한정** 규격만 다루며, 공통 규칙(컬러 사용·접근성 대비·심의 항목 등)은 상위 가이드를 따른다.

---

## 1.1 한 줄 정의

> **"알바몬 PC 메인 좌측 상단, BI 바로 옆에 붙어 있는 가로형 1슬롯 광고 배너. 메인카피 1줄 + 서브카피 1줄 + (선택) 보조 비주얼 + 출처표기 영역으로 구성된다."**

이 배너는 알바몬 메인을 처음 본 사용자의 시선이 BI 다음으로 닿는 자리다. 카피 한두 줄로 광고주의 메시지를 전달하고, 클릭으로 광고주 랜딩으로 이어지는 단일 전환 슬롯이다. 입고는 1556×232 한 장이지만 화면에는 778×116(2x retina)로 축소 노출된다.

---

## 1.2 사용 컨텍스트

| 사용처 | 적합 여부 | 이유 |
|--------|----------|------|
| 알바몬 메인 좌측 상단 | ✅ | 본 가이드의 정의 위치 |
| 알바몬 메인 우측 / 하위 영역 | ❌ | 사이즈·여백·폰트 규격이 다르므로 해당 영역 가이드 별도 |
| 알바몬 외 매체(잡코리아 등) | ❌ | 매체별 BI·배경 정책이 달라 별도 규격 |
| 모바일 메인 탑 배너 | ❌ | 모바일은 별도 가이드 (해상도·폰트 토큰 별도) |

> **PC 메인 좌측 = BI 인접 슬롯**. BI(붉은 계열 캐릭터·로고)와 시각적으로 충돌하면 사용자가 광고를 BI의 일부로 오인한다. §1.5 배경 컬러 정책이 본 가이드에서 가장 깐깐한 항목인 이유다.

---

## 1.3 노출 스펙

| 속성 | 값 |
|------|-----|
| 노출 위치 | PC > 알바몬 메인 좌측 상단 |
| 입고 사이즈 | **1556 × 232 px** (2x) |
| 화면 내 노출 사이즈 | **778 × 116 px** (1x) |
| 파일 크기 | **50KB 이내** |
| 파일 유형 | `jpg`, `jpeg`, `gif`, `png` |
| 마감 처리 | 라운딩(모서리 둥글림) **없이** 직각 사각형으로 전달 — 라운딩은 매체에서 처리 |

> **2x 입고의 이유**: 고해상도 디스플레이에서 폰트·디테일이 깨지지 않도록 입고는 항상 2x. 디자인 작업도 1556×232 캔버스에서 시작하고, 가독성 검수는 778×116 축소 상태에서 한 번 더 본다.
> **GIF 사용 시**: 자동재생·반복 횟수·페이즈별 가독성은 공통 가이드의 모션 정책을 따른다. 메인카피가 한 페이즈에서만 잡히면 가독성이 깨지므로, 모든 페이즈에서 메인카피의 가독이 유지되어야 한다.

## 1.4 레이아웃 — 여백·슬롯

```
┌── 입고 캔버스 1556 × 232 ──────────────────────────────────────────────────┐
│ ↕ 54px (상)                                                  ↕ 16px (출처)  │
│ ┌──────────────────────────────────────────────────────────────────────┐  │
│ │                                                                ┌───┐ │  │
│ │                                                                │출처│ │  │
│ │  ─ 96px ─  ┌── 카피 영역 ────────────────┐  ┌── 보조 비주얼 ─┐  └───┘ │  │
│ │            │ 서브카피 1줄 (Paperlogy 4)   │  │ (선택, 우측)   │        │  │
│ │            │ 메인카피 1줄 (Paperlogy 7)   │  │ ALBAMON / 캐릭 │        │  │
│ │            └─────────────────────────────┘  └────────────────┘        │  │
│ │                                                                ─ 96px ─│  │
│ └──────────────────────────────────────────────────────────────────────┘  │
│ ↕ 54px (하)                                                                 │
└────────────────────────────────────────────────────────────────────────────┘

좌·우 안전 여백:  96px
상·하 안전 여백:  54px
출처 영역 여백:   16px (캔버스 우측 상단 기준)
```

| 슬롯 | 값 |
|------|-----|
| 좌 안전 여백 | 96px |
| 우 안전 여백 | 96px |
| 상 안전 여백 | 54px |
| 하 안전 여백 | 54px |
| 출처표기 여백 | 16px (우측 상단 권장) |
| 카피 영역 | 좌측 정렬, 메인 1줄 + 서브 1줄 |
| 서브↔메인 간격 | 19px |
| 보조 비주얼 영역 | 우측(선택) — 캐릭터·제품 컷 등 |

> **여백은 "안전 영역"**: 96/54px 안쪽으로 카피·핵심 비주얼이 들어가지 않는다. 배경 컬러·패턴은 캔버스 가장자리까지 채워도 되지만, 카피와 식별 요소는 안전 여백 안에서만 배치한다.
> **출처표기 위치**: 우측 상단을 권장한다. 좌측 상단·하단으로 옮기면 카피 동선과 충돌하므로 사용 컨텍스트가 분명할 때만 변경한다.

## 1.5 배경 컬러 정책

알바몬 BI(주황·붉은 계열 캐릭터/로고)와 인접한 자리이므로 배경 컬러 선택이 가장 큰 제약이다.

| 항목 | 정책 |
|------|------|
| 붉은 계열 배경 | **지양** (BI와 시각적 동화 발생) |
| 배경 ↔ 알바몬 메인 페이지 | 구분이 되는 색을 사용 |
| 이미지 외곽 테두리 | **삽입 불가** (테두리는 매체에서 컨트롤) |

> **"배경과 구분되는 Background Color"의 의미**: 알바몬 메인 페이지 배경(흰색·연회색 톤)과 배너가 섞여 보이지 않도록, 배너 자체의 배경에 한 단계 톤을 잡는다. 단색 배경이든 그라데이션이든 무방하지만, 배너 사각형의 경계가 페이지에서 인식돼야 한다.
> **테두리 삽입 금지**: 배너 외곽에 1px 라인을 그어 "박스를 그리는" 처리는 매체 표준과 충돌한다. 외곽 인식은 §1.5 배경 컬러로 해결할 것.
> **붉은 계열을 꼭 써야 한다면**: BI와 다른 채도·명도로 충분히 차이를 두거나, 카피 영역 뒤에만 부분적으로 사용. 캔버스 전체를 BI와 같은 톤의 붉은색으로 깔지 말 것.

---

## 1.6 카피 슬롯

배너 안에 들어가는 카피는 **서브카피 1줄 + 메인카피 1줄** 구조를 권장한다.

```
서브카피 (작게, 위)        ← 광고 노출할 곳을 찾는다면?
메인카피 (크게, 아래)      ← 지금 바로 알바몬에 광고하세요
```

| 슬롯 | 권장 |
|------|------|
| 서브카피 | 1줄 (앞·맥락·질문형 카피) |
| 메인카피 | 1줄 (행동 유도·핵심 메시지) |
| 정렬 | 좌측 정렬 권장 |
| 줄 수 | 메인 1줄 / 서브 1줄 (2줄 합산) |

> **2줄 합산 권장의 이유**: 778×116로 축소되면 줄 수가 늘수록 가독이 빠르게 떨어진다. 메시지를 두 단계(맥락→행동)로 끊어 한 줄씩 배치하는 패턴이 검증된 기본형.
> **카피가 길어질 때**: 2줄 권장을 깨야 한다면 서브카피를 더 짧게 줄여 메인카피의 자릿수를 확보. 메인카피는 줄바꿈 금지가 기본.
> **광고주가 직접 카피를 결정**: 본 가이드는 슬롯 형태만 정의한다. 카피 워딩 자체의 톤은 매체별 광고 심의 가이드에 따른다.

---

## 1.7 출처 표기

| 속성 | 값 |
|------|-----|
| 위치 | 캔버스 우측 상단 권장 |
| 여백 | 우측 16px / 상단 16px |
| 표기 형식 | 공통 가이드 준수 |

> **광고주 ↔ 매체 구분**: 출처 표기는 광고주가 누구인지를 명시하는 영역이다. 메인카피 안에 광고주명을 녹였더라도 출처 표기는 별도로 둔다.

---

## 1.8 폰트 토큰

| 속성 | 메인 폰트 | 서브 폰트 |
|------|----------|----------|
| 폰트 패밀리 | Paperlogy | Paperlogy |
| 굵기 | 7 Bold | 4 Regular |
| 크기 | 62px | 38px |
| 자간(letter-spacing) | -4% | -2% |
| 용도 | 메인카피 (행동 유도) | 서브카피 (맥락) |

| 공통 제약 | 값 |
|-----------|-----|
| 최소 사이즈 | 26px (어떤 슬롯이든 이 이하로 내리지 않는다) |

> **62/38 비율의 의미**: 메인이 서브 대비 약 1.6배. 778×116 축소 상태에서도 메인이 한눈에 들어오는 비율. 자간을 음수로 잡는 이유는 Paperlogy의 기본 자간이 다소 넓어 큰 사이즈에서 글자가 흩어져 보이기 때문.
> **26px 미만 금지**: 출처 표기·법적 고지 같은 텍스트도 26px 아래로 내려가면 778×116에서 판독이 안 된다. 길어질 카피가 26px 미만이 될 것 같으면 카피 자체를 줄인다.
> **Paperlogy 외 폰트 사용**: 광고주 BI 폰트가 별도로 있을 경우 메인카피 한정으로 광고주 폰트 사용 가능. 본 가이드의 사이즈·자간 규칙은 그대로 따른다.

## 1.9 마감 규칙

| 항목 | 규칙 |
|------|------|
| 라운딩 | 최종 파일은 **라운딩 없이** 직각 사각형으로 전달 (라운딩은 매체에서 적용) |
| 외곽 테두리 | 삽입 불가 |
| 파일 용량 | 50KB 이내 |
| 파일 유형 | jpg, jpeg, gif, png |
| 색공간 | sRGB |
| 입고 사이즈 | 1556 × 232 (2x) |

> **라운딩을 매체에서 처리하는 이유**: 매체 디자인이 변경되면 라운딩 반경도 바뀐다. 광고주 소재에 라운딩이 미리 그어져 있으면 매체 라운딩과 1~2px 어긋나 어색해 보인다.
> **50KB 제한 대응**: 단색 배경은 jpg가 아닌 png가 더 가벼운 경우가 있다. GIF로 모션을 넣는다면 프레임 수·팔레트 256색 이하로 압축. 50KB 초과 시 압축 우선이며, 사이즈/해상도를 낮추지 말 것 (해상도 낮추면 카피 가독이 무너진다).

## 1.10 검수 체크리스트

> 입고 직전에 한 번씩 확인. 하나라도 어긋나면 매체에서 반려된다.

- [ ] 입고 사이즈가 1556 × 232인가
- [ ] 778 × 116으로 축소했을 때 메인카피·서브카피가 모두 판독되는가
- [ ] 좌우 96px / 상하 54px / 출처 16px 안전 여백 안에 핵심 요소가 들어가 있는가
- [ ] 배경이 BI(붉은 계열)와 충돌하지 않는가
- [ ] 알바몬 메인 페이지 배경과 배너 배경이 시각적으로 구분되는가
- [ ] 외곽 1px 테두리가 들어가 있지 않은가
- [ ] 모서리에 라운딩이 적용되어 있지 않은가
- [ ] 메인카피가 1줄 / 서브카피가 1줄을 지키는가
- [ ] 모든 텍스트가 26px 이상인가
- [ ] 메인 = Paperlogy 7 Bold 62px / -4% / 서브 = Paperlogy 4 Regular 38px / -2% 인가
- [ ] 출처표기가 우측 상단(16px 안쪽)에 있는가
- [ ] 파일 용량이 50KB 이내인가
- [ ] 파일 유형이 jpg / jpeg / gif / png 중 하나인가
- [ ] (GIF인 경우) 모든 페이즈에서 메인카피의 가독이 유지되는가

## 1.11 자주 하는 실수

| 실수 | 결과 | 해결 |
|------|------|------|
| 붉은 톤 배경을 캔버스 전체에 깔음 | BI와 동화되어 사용자가 광고를 인지하지 못함 | §1.5 — 다른 톤으로 변경하거나 부분 사용 |
| 메인카피 줄바꿈 | 778×116에서 메인이 흩어져 한눈에 안 들어옴 | 카피를 한 줄로 줄이거나 서브로 분리 |
| 외곽 1px 테두리 | 매체 라운딩과 어긋나 어색해짐 | 테두리 제거, 배경 톤으로 외곽 인식 |
| 모서리 라운딩 미리 적용 | 매체 라운딩과 이중 적용되어 깨짐 | 직각 사각형으로 전달 |
| 26px 미만 작은 텍스트(법적고지 등) | 축소 후 판독 불가 | 카피 자체를 짧게 다듬어 26px 이상 확보 |
| 50KB 초과 GIF | 매체 반려 | 프레임 수·팔레트 압축 우선, 해상도는 유지 |
| 1x(778×116)로 입고 | 고해상도에서 폰트 깨짐 | 무조건 2x(1556×232)로 입고 |

## 1.12 참조

| 항목 | 위치 |
|------|------|
| 배너 공통 제작가이드 | 별도 문서 (컬러·접근성·심의 공통 규칙) |
| 출처 표기 형식 | 공통 가이드 |
| 알바몬 메인 페이지 BI | 매체 디자인 시스템 |
| 모바일 메인 탑 배너 | 별도 가이드 (모바일은 해상도·폰트 토큰 분리) |

> 다음 작업: 모바일 메인 탑 배너, 그 외 알바몬 메인 하위 영역 배너의 가이드도 같은 포맷으로 분리해 둘 것.

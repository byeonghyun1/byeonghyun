# 디자인 공통 규칙 가이드

> ⚠️ **이 문서는 사람용 보조 뷰입니다.** 데이터의 진실(SSoT)은 `config/design_rules.json`. 어긋날 경우 JSON이 우선합니다. 수정은 JSON에서.

> 최종 업데이트: 2026-04-27

---

## 공통 규칙

### 1. 파일

- 배너 공통 제작가이드 파일명 규칙 준수
- 허용 확장자: jpg, jpeg, gif, png
- 기본 최대 파일 크기: 30KB
- 출처 표기: 8px, 고딕 계열 폰트

---

### 2. 폰트

- **고딕 계열만 사용** 가능
- 폰트 스타일 최대 **2가지**까지
- PC 최소 폰트 사이즈: **12px**
- 모바일 최소 폰트 사이즈: **22px**
- 미세한 텍스트 배경 처리 가능 (가시성 확보용)
- 배경과 텍스트 간 **충분한 명도 대비** 필수

---

### 3. 카피 (문구)

- 텍스트 컬러 최대 **2가지**
- 폰트 사이즈에 맞는 적절한 행간 유지
- 간결하고 명확한 카피 사용 권장

**Good Case:**
- 간결하고 명확한 문구
- 핵심 키워드 강조

**Bad Case:**
- 너무 긴 문장
- 복잡한 문구 구성

---

### 4. 이미지

- 고해상도 이미지 사용 필수
- 깔끔한 누끼(배경 제거) 처리
- 전면 사진 배경 지양
- 이미지 외곽에 테두리 삽입 불가 (일부 배너 제외)

---

### 5. 컬러

- 단색(솔리드) 배경 권장
- 패턴 배경 지양
- 화이트 단독 배경 지양 (가시성 있는 BG컬러 사용)
- 네온/형광 컬러 사용 금지
- 붉은계열 배경은 BI 인접 배너에서 지양

---

### 6. 레이아웃

- 모바일 배너: 10px 유연 영역 확보
- 각 배너별 지정된 여백(margin) 엄수
- 출처 표기 영역 별도 확보

---

## 스플래시 배너 전용 규칙 (m_app_splash)

### Good Case

| 유형 | 규칙 |
|---|---|
| 로고 + 오브젝트 | 문구/로고 영역을 준수하며 광고 브랜드 인지 가능 |
| 문구 + 오브젝트 | 문구/로고 영역을 준수하며 오브젝트 최소 여백 준수 |
| 문구2줄 + 로고 + 오브젝트 | 문구 영역 준수, 브랜드 로고 최소 사이즈 준수, 브랜드 광고 내용 인지 가능 |

### Bad Case

| 유형 | 문제점 |
|---|---|
| 광고 오브젝트만 노출 | 광고 이미지 좌우 여백을 고려하지 않음 |
| 로고 + 오브젝트 영역 미준수 | 문구/로고 영역을 준수하지 않은 브랜드 로고 사이즈 |
| 문구 + 오브젝트 영역 미준수 | 오브젝트 최소사이즈 (880px) 미준수하여 과하게 작음 |

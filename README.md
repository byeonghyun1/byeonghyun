## 프로젝트 개요

이 프로젝트는 **AI 콘텐츠 디자인 에이전트 시스템**을 구축하는 것을 목표로 합니다.

### 최종 목표
자연어 입력(바이브 코딩)으로 카피, 이미지, 브랜드 스타일을 자동 반영하여 디자인 결과물을 생성하는 지능형 시스템 엔진.

### 주요 성과
- **card_news_generator** 프로토타입 완성 (1차 목표 달성)
- 카드뉴스, 배너, 포스터 3개 콘텐츠 유형 기본 구현
- 3개 브랜드(알바몬, 잡코리아, 워스피어) 스타일 프리셋 구축
- HTML 템플릿 기반 렌더링 + Anthropic API 연동 완료
- SVG/PNG 다중 포맷 출력 지원

### 기본 통계
- **전체 메시지:** 131개
- **사용자 질문:** 32개
- **어시스턴트 응답:** 99개
- **활용한 도구:** 11가지 (파일 읽기, 쓰기, 코드 분석, 구조 설계 등)

## 핵심 아키텍처

### 3축 구조 (3-Axis Architecture)

```
콘텐츠 유형 (Content Type)
├── card_news (카드뉴스)
├── banner (배너)
└── poster (포스터)

브랜드 (Brand)
├── albamon (알바몬)
├── jobkorea (잡코리아)
└── workspia (워스피어)

사이즈 (Size)
├── 1080x1080 (카드뉴스)
├── 1200x628 (배너)
└── A4_vertical (포스터)
```

### 데이터 흐름

```
입력 (자연어/바이브 코딩)
  ↓
AI 카피 생성 (Anthropic API / 더미 생성기)
  ↓
레이어 트리 빌드 (content_types/[type]_layers.py)
  ↓
SVG 렌더링 (engine/svg_exporter.py)
  ↓
PNG 변환 (imagemagick / playwright)
  ↓
결과물 Export (SVG / PNG / HTML)
```

### 주요 모듈

| 모듈 | 역할 |
|---|---|
| `engine/` | 핵심 엔진 (카피 생성, SVG 내보내기, 색상 관리) |
| `content_types/` | 콘텐츠 유형별 구현 + 레이어 빌더 |
| `brands/` | 브랜드별 스타일 정의 |
| `templates/` | HTML 템플릿 (참고용) |

## 주요 설계 결정

### 1. 브랜드 관리 전략
- **선택:** 통합 에이전트 + 스타일 옵션 선택
- **이유:** 확장성 높음, 새 브랜드 추가 용이

### 2. 렌더링 방식
- **선택:** HTML 파싱 대신 LayerTree 직접 생성
- **이유:** Figma 호환성, 텍스트 편집 가능성 보장

### 3. 카피 생성 폴백
- **API 실패 시:** 더미 카피 자동 생성
- **이유:** 프로토타입 개발 속도 향상, 안정성

### 4. 다중 포맷 출력
- **SVG:** Figma 편집용
- **PNG:** 최종 결과물용
- **HTML:** 디버깅용
